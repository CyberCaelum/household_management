# 微信支付扫码支付流程说明

## 整体流程

```
┌─────────────┐         ┌─────────────┐         ┌─────────────┐
│   雇主APP   │ ──────> │  你的后端   │ ──────> │  微信支付   │
│  (扫码支付) │         │             │         │             │
└─────────────┘         └─────────────┘         └─────────────┘
       │                       │                       │
       │                       │                       │
       │                       │                       │
       │                       │                       ▼
       │                       │              ┌─────────────┐
       │                       │              │  用户扫码   │
       │                       │              │  完成支付   │
       │                       │              └─────────────┘
       │                       │                       │
       │                       ▼                       │
       │              ┌─────────────┐                  │
       │              │ 回调 /notify │ <───────────────┘
       │              │    /pay     │
       │              └─────────────┘
       │                       │
       │                       ▼
       │              ┌─────────────┐
       └────────────> │  订单状态   │
                      │   已支付    │
                      └─────────────┘
```

## 接口说明

### 1. 获取支付二维码

**接口**: `PUT /order/payment`

**请求参数**:
```json
{
  "orderId": 123,
  "orderNumber": "ORDER202403170001",
  "payMethod": 1
}
```

**响应**:
```json
{
  "code": 1,
  "msg": "操作成功",
  "data": "weixin://wxpay/bizpayurl?pr=xxxxxx"
}
```

**前端处理**:
```javascript
// 1. 调用支付接口获取二维码URL
const res = await request.put('/order/payment', {
  orderId: orderId,
  orderNumber: orderNumber,
  payMethod: 1  // 微信支付
});

// 2. 使用 qrcode.js 等库生成二维码图片
const codeUrl = res.data;
QRCode.toCanvas(document.getElementById('qrcode'), codeUrl);

// 3. 开始轮询查询订单支付状态
startPolling(orderNumber);
```

---

### 2. 支付回调接口（微信调用）

**接口**: `POST /notify/pay`

**说明**: 这是微信调用的接口，**不需要前端调用**。微信在用户支付成功后会主动推送通知到这个接口。

---

### 3. 查询订单状态（前端轮询）

建议添加一个查询订单状态的接口供前端轮询：

```java
@GetMapping("/payStatus/{orderNumber}")
public Result<Integer> getPayStatus(@PathVariable String orderNumber) {
    Order order = orderMapper.getOrderByNumber(orderNumber);
    if (order == null) {
        return Result.error("订单不存在");
    }
    return Result.success(order.getPayStatus());
}
```

**前端轮询示例**:
```javascript
function startPolling(orderNumber) {
  const timer = setInterval(async () => {
    const res = await request.get(`/order/payStatus/${orderNumber}`);
    
    if (res.data === 1) {  // 已支付
      clearInterval(timer);
      alert('支付成功！');
      // 跳转到订单详情或列表页
      router.push('/orders');
    }
  }, 2000);  // 每2秒查询一次
  
  // 5分钟后停止轮询
  setTimeout(() => {
    clearInterval(timer);
  }, 5 * 60 * 1000);
}
```

## 配置说明

### 1. 配置回调地址

在 `application.yml` 中配置：

```yaml
wechat:
  pay:
    app-id: wx1234567890abcdef
    mch-id: 1234567890
    api-v3-key: your-api-key
    mch-serial-no: your-cert-serial
    private-key-path: /path/to/apiclient_key.pem
    notify-url: https://your-domain.com/notify/pay  # ⚠️ 必须是外网可访问的HTTPS地址
```

### 2. 内网穿透（开发环境）

如果是本地开发，需要使用内网穿透工具让微信能访问到你的回调接口：

**使用 ngrok**:
```bash
# 安装 ngrok
# 注册并获取 authtoken
ngrok config add-authtoken your-token

# 启动穿透（假设你的后端运行在 8080 端口）
ngrok http 8080

# 会生成一个 https://xxxx.ngrok-free.app 的地址
# 将这个地址配置到 notify-url: https://xxxx.ngrok-free.app/notify/pay
```

**使用花生壳**:
- 注册花生壳账号
- 添加内网映射，指向本地 8080 端口
- 使用分配的域名配置 notify-url

## 安全说明

1. **签名验证**: `WechatPayUtil.parsePayNotify()` 会自动验证微信的签名，确保通知是微信发送的

2. **幂等性处理**: `paySuccess` 方法已添加幂等性检查：
   - 如果订单已经支付（`payStatus = 1`），直接返回
   - 防止同一订单重复处理

3. **状态校验**: 只有待支付状态的订单才会处理支付成功

4. **响应要求**: 必须返回成功响应给微信，否则微信会重复通知（最多8次）

## 常见问题

### Q1: 用户支付后订单状态没有更新？
- 检查 `notify-url` 是否配置正确，必须是外网可访问的 HTTPS 地址
- 检查日志是否有回调请求进来
- 检查签名验证是否通过（查看错误日志）

### Q2: 微信重复通知？
- 确保回调接口返回了成功响应（`buildNotifySuccessResponse()`）
- 检查接口是否有异常抛出

### Q3: 如何处理支付超时？
- 可以在创建订单时设置支付超时时间
- 使用定时任务关闭超时未支付的订单
- 或者让用户在前端主动取消订单

### Q4: 如何测试支付？
- 使用微信沙箱环境（配置 `sandbox: true`）
- 或者使用微信开发者工具的支付调试功能
- 注意：微信支付的测试必须使用真实的微信客户端扫码
